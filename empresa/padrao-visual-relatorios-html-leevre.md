# Padrão Visual — Relatórios HTML Leevre

## Objetivo
Padronizar relatórios HTML gerados pela Livri para que sejam:
- claros
- rápidos de ler
- visualmente consistentes
- úteis para decisão

## Formato recomendado
Arquivo HTML standalone com:
- dark theme
- abertura direta no browser
- layout responsivo
- leitura rápida no desktop e mobile

## Direção visual
### Tema
- fundo principal escuro
- referência inicial: `#0A0E2A`

### Tipografia
- fonte limpa e moderna
- referência aprovada: Inter

## Componentes prioritários
### 1. KPIs em cards coloridos
Usar cards com destaque visual para leitura imediata.

### 2. Barra horizontal de urgência
Boa para mostrar:
- volume de pendências
- risco operacional
- temperatura comercial
- status de metas

### 3. Tabelas com badges coloridos
Boa para:
- urgência
- prioridade
- status
- risco

### 4. Estrutura de leitura
Relatório deve seguir esta ordem:
1. resumo executivo
2. KPIs principais
3. alertas
4. tabela operacional
5. recomendações
6. próximos passos

## Onde isso faz mais sentido na Leevre
### Operação
- comissão
- conciliação
- pendências
- divergências

### Comercial
- pipeline
- leads esfriando
- negociações abertas
- perdas por motivo

### Marketing
- campanhas
- criativos
- origem dos leads
- qualidade por canal

## Regra
Visual bonito sem priorização clara vira enfeite.
O HTML da Leevre deve ajudar a decidir rápido.
