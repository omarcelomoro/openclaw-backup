# Rotina — revisão de pendências da Livri

## Objetivo
Evitar que trabalho importante suma no chat ou fique sem conclusão.

## Quando usar
- quando Marcelo pedir o que está pendente
- no fim de blocos relevantes de trabalho
- quando houver itens aguardando retorno, arquivo, acesso ou decisão

## Fontes principais
- `memory/pending.md`
- `memory/context/pending.md`
- arquivos de projeto em `memory/projects/`
- memória do dia, quando houver contexto novo ainda não consolidado

## Saída padrão para o Marcelo
Separar em 4 blocos curtos:
1. **Prioridade agora**
2. **Aguardando Marcelo**
3. **Aguardando terceiros/sistema**
4. **Itens parados / depois**

## Regra de manutenção
Se um item reaparecer mais de uma vez, ele deve ganhar contexto mínimo:
- por que está parado
- o que falta para destravar
- qual é o próximo passo concreto

## Critério
Lista curta, viva e útil.
Não virar depósito de ideias soltas.
